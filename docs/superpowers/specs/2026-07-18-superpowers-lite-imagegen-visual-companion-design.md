# Superpowers Lite ImageGen 视觉伴侣设计

## 背景

`brainstorming` 当前通过本地 HTML、HTTP/WebSocket 服务和浏览器事件提供视觉伴侣。它能表达
布局，但也引入 5 个运行脚本、临时会话、端口、会话密钥、浏览器可达性和清理逻辑。对 Codex
与 ChatGPT 用户而言，这些基础设施并不是视觉讨论的核心，还可能因环境或页面实现质量分散注意力。

本次将视觉伴侣改为 Codex/ChatGPT 内置 ImageGen：需要比较界面、视觉风格、产品概念或空间
构图时，直接生成高保真图片并在对话中内联展示。用户已明确不需要 CLI、API Key、模型降级、
HTML 或其他回退路径。

## 设计原则

1. **ImageGen-only：** 只使用当前 Codex/ChatGPT 提供的内置 ImageGen 能力，不调用外部 API，
   不提供 CLI 或模型降级路径。
2. **图片解决视觉问题：** UI 布局、视觉风格、产品形态和多方向对比优先生成图片；范围、API、
   数据模型、精确流程和文字取舍继续在对话中处理。
3. **精确内容不押在位图上：** 关键文案、数字、接口和验收条件始终在图片旁用文字重述。需要
   精确技术图时使用 Mermaid、表格或代码原生表达，不用 HTML 视觉伴侣，也不让生成图成为唯一
   事实来源。
4. **统一比较变量：** A/B/C 方案使用相同视口、内容密度和数据，只改变当前要决策的主要变量，
   避免把布局、配色和信息架构同时换掉。
5. **渐进迭代：** 首图用于建立方向；后续以已生成图片为参考，一次只改变一个主要变量，并重复
   不得漂移的约束。
6. **资产边界清楚：** 纯头脑风暴预览只需内联展示；一旦图片被批准并由项目引用，必须复制到
   工作区的明确路径，不能只留在 Codex 的默认生成目录。

## 交互流程

1. 判断当前开放决策是否真的依赖视觉判断；若文字足够，不生成图片。
2. 告知用户图片将澄清什么，然后组织简短生成规格：用途、画布、主体、布局、风格、选项标签、
   精确文字、必须保持和避免事项。
3. 使用内置 ImageGen 生成单个方案或一张包含 2～4 个方向的对比图，并直接内联展示。
4. 在图片旁以文字列出 A/B/C 的核心差异与关键精确信息，请用户回复选择和修改意见。
5. 反馈需要改图时，使用上一张图作为编辑目标或参考，只做一个聚焦修改后重新展示。
6. 用户批准后，把决定写回设计结论；若图片成为项目资产，再保存到工作区并报告路径。

## 失败语义与恢复

- 内置 ImageGen 不可用或生成失败时，明确说明本轮无法生成视觉稿，保留已经确认的文字决策；
  不启动本地服务，不切换 CLI/API，也不声称已经展示成功。
- 图片中文字不准确时，以对话中的精确文字为准；必要时减少图中文字并重新生成。
- 多方案比较失去可比性时，统一输入、视口和风格后重做，不能从混杂变量的图片直接得出结论。
- 迭代发生视觉漂移时，重新指定上一版参考图与不变量，撤销无关变化。

## 验收标准

- `brainstorming` 明确使用内置 ImageGen、内联展示、方案比较、精确文字旁路和项目资产保存边界。
- 视觉伴侣正文不包含 HTML 服务、浏览器会话、CLI、API Key、模型降级或其他回退流程。
- `skills/brainstorming/scripts/` 不存在，打包归档不再包含旧视觉伴侣运行代码。
- 压力场景能阻止混杂变量比较、位图承载唯一精确信息以及偷偷切换回退路径。
- 版本三处一致为 `0.1.4`；Node 契约、技能校验、插件校验、运行脚本和打包测试全部通过。
- 每个独立交付验证后立即形成 Git 提交；未授权推送。
