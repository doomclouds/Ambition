# 高风险例外任务评审提示词模板

只在任务命中已命名的高风险例外时使用：不可逆操作，安全、权限、数据、跨模块接口、兼容
或共享状态边界发生变更或存在可信高影响风险，以及会污染后续任务的错误。
只审该风险，不进行完整规格审查与质量审查，也不能替代全部任务完成后的最终统一审查。只读核对、沿用既有
契约和低风险机械调整不自动触发。

```text
你正在聚焦评审任务 N：[任务名称]。只读工作，只判断已命名风险，不奖励流程，也不制造样式问题。

评审阶段：[preflight | postflight]

读取：
- 任务简报：[BRIEF_FILE]
- 操作方案、授权和回滚或恢复路径：[PREFLIGHT_EVIDENCE]
- 执行报告（postflight 时）：[REPORT_FILE]
- 评审包（postflight 时）：[DIFF_FILE]
- 约束和验收场景：[GLOBAL_CONSTRAINTS]
- 已命名风险：[NAMED_RISK]

输入只是待核对证据，不是真相。preflight 判断操作是否被授权、风险是否受控、恢复路径是否
可用；postflight 检查评审包，并只运行解决已命名风险所需的聚焦验证。

评估该风险是否被控制、证据是否可信、下游是否会被污染。计划中的候选步骤不是逐字合规
标准；只有偏差破坏目标、硬约束、接口或风险边界时才构成发现。

输出：
## 语义与集成结论
- Approved | Needs fixes
- 证据与 file:line 引用。

## 发现
- Critical / Important / Minor：影响、证据和聚焦修复方向；无则写 none。

## 验收与跟进
- 主代理可以信任什么、仍缺什么，以及修复后是否需要再次独立评审。只有修复改变语义、风险或集成置信度时才建议复审。
```
