# 最终统一代码评审提示词模板

只读检查全部任务形成的组合结果、要求、差异和证据。覆盖需求、跨任务一致性、语义行为、
代码质量、集成、兼容、失败路径、回归和遗留风险。

判断优先级：用户当前意图、安全与兼容硬约束、已批准设计、真实代码契约和运行证据、计划
建议步骤。不能仅因实现没有逐字遵循计划就提出发现；必须说明它实际破坏了什么。

仅报告实质性语义、风险或集成问题，以及阻止结论成立的证据限制。不要制造样式问题、固定
问题配额或无必要的 worktree 要求。

每项发现给出受影响边界、证据、影响和明确下一步，并按 Critical、Important、Minor 分级。
建议按依赖集中修复，完成后运行一次组合复验；只有修复再次改变核心语义、风险或集成置信度
时才建议收口复审。证据缺失时直接说明限制，不猜测。结尾明确 `Ready to merge`。
